# Аналіз та Проєктування Голосового Бота (Завдання 1)

## Документація: "Голосовий бот інтерактивного кулінарного помічника"

Цей документ містить аналіз та архітектуру голосового бота, що відповідає вимогам Завдання 1, з акцентом на кулінарного помічника (Варіант 14).

### 1. Збір даних предметної області
*   **Мета бота:** Надання інтерактивної допомоги користувачеві під час приготування страв на основі наявних інгредієнтів. Бот має сприймати голосові команди, пропонувати рецепти та інструктувати під час готування у реальному часі.
*   **Функціональні вимоги:**
    *   Розпізнавання голосу користувача (Speech-to-Text).
    *   Генерація голосових відповідей (Text-to-Speech).
    *   Розпізнавання намірів користувача (пошук рецепту, заміна інгредієнтів, наступний крок тощо).
    *   Динамічне оновлення списку інгредієнтів (наприклад, "я замість вершкового масла використовую кокосову олію").
    *   Інтеграція з месенджером (Telegram).
*   **Нефункціональні вимоги:**
    *   Швидкість відповіді не більше 3-5 секунд.
    *   Висока надійність розпізнавання мови навіть з шумом (кухня).
    *   Підтримка української мови.
*   **Цільова аудиторія:** Люди, які готують вдома і мають зайняті руки, що унеможливлює користування телефоном під час приготування.
*   **Сценарії використання:** Користувач перелічує інгредієнти -> бот пропонує рецепт -> користувач погоджується -> бот диктує кроки. Під час приготування користувач може змінити інгредієнт -> алгоритм перелаштовується.

### 2. Проєктування архітектури
*   **Компоненти:**
    1.  **Інтерфейс користувача (Месенджер):** Telegram для обміну голосовими повідомленнями.
    2.  **Модуль Speech-to-Text (ASR):** Groq API (модель `whisper-large-v3`) для швидкого та точного транскрибування.
    3.  **Модуль NLP та ШІ:** Google Gemini (`gemini-2.5-flash`) для розуміння намірів, генерації рецептів та пам'яті (відстеження кроків та інгредієнтів).
    4.  **Модуль Text-to-Speech (TTS):** Бібліотека `gTTS` для синтезу мови.
*   **Джерела даних:** Динамічні знання LLM Gemini щодо існуючих рецептів і кулінарних прийомів.
*   **Архітектурна схема:**
    `Користувач -> Telegram Bot -> (Завантаження Voice) -> Groq Whisper (Text) -> LLM Engine Gemini (Відповідь + Контекст) -> gTTS (Audio) -> Telegram Bot -> Користувач`

### 3. Розробка діалогових сценаріїв
1.  **Привітання користувача:**
    *   *Бот:* "Привіт! Я ваш інтерактивний кулінарний помічник. Які інгредієнти у вас є?"
2.  **Пошук рецепту / Динамічні зміни:**
    *   *Користувач (Голос):* "У мене є курка, картопля, але замість вершкового масла - кокосова олія."
    *   *Бот:* "Чудово, ми можемо приготувати запечену курку з картоплею на кокосовій олії. Розпочати?"
3.  **Інструктаж по кроках:**
    *   *Користувач:* "Так, що робити спочатку?"
    *   *Бот:* "Крок 1: Наріжте картоплю... Готові до наступного кроку?"
4.  **Завершення діалогу:**
    *   *Користувач:* "Дякую, страва готова."
    *   *Бот:* "Смачного! Звертайтесь ще, коли зголоднієте."
*   **Діаграма станів:** `[Початок] -> [Привітання] -> [Збір інгредієнтів] -> [Пропозиція рецепту] -> [Покрокове приготування] -> [Завершення]` (Можливість повернення на попередні кроки при зміні інгредієнтів).

### 4, 5. Базова логіка та Інтеграція ШІ
*   **ASR:** Використання Groq API (Whisper) забезпечить найвищу швидкість і точність розпізнавання, що ідеально підходить для голосового бота.
*   **TTS:** `gTTS` застосовується для створення голосових MP3-файлів з тексту, згенерованого моделлю ШІ.
*   **NLP/Інтеграція ШІ:** Замість класичного Dialogflow ми використаємо *LLM (Gemini 2.5 Flash)*, що дозволяє значно гнучкіше підтримувати контекст діалогу. Специфіка "кулінарного помічника" вимагає динамічного реагування на такі події, як заміна інгредієнтів чи уточнення кроку. Контроль намірів здійснюватиметься спеціалізованим системним промптом (System Instruction), який формує поведінку бота як експерта-кулінара.

### 6, 7, 8. Тестування, Розгортання, Вдосконалення
*   **Тестування:** Модульне тестування буде реалізовано локально через CLI скрипт для перевірки ASR, TTS та динаміки контексту LLM.
*   **Розгортання:** Бот працюватиме через довготривале опитування (Long Polling) за допомогою бібліотеки `aiogram` для месенджера Telegram.
*   **Оцінка:** Після запуску передбачається збір логів розпізнавання: на випадках, коли Whisper помиляється у розпізнаванні специфічних кулінарних термінів або Gemini дає задовгі відповіді, промпти та конфігурація модулів будуть доопрацьовуватись для оптимізації.
